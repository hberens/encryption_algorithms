AES description

Type: Symmetric block cipher (substitution-permutation network; byte-oriented).

How it works AES uses 128-bit blocks arranged as a 4×4 state of bytes. Before rounds, the state is XORed with the first four words of the key schedule. Each encryption round (except the last) applies, in order: SubBytes (byte substitution via S-box / multiplicative inverse in GF(2⁸) plus affine map), ShiftRows (row i shifted by i bytes), MixColumns (mix each column in GF(2⁸) using the 02/03 matrix), and AddRoundKey (XOR four schedule words). The last encryption round omits MixColumns. Decryption uses the inverse steps in the order Avi Kak gives: InvShiftRows, InvSubBytes, AddRoundKey, InvMixColumns, with the last decryption round omitting InvMixColumns.

Rounds: 10 for 128-bit keys, 12 for 192-bit, 14 for 256-bit. Key expansion extends the key into round keys.

Key / block size: Key 128, 192, or 256 bits; block size always 128 bits.

Use case: Wi-Fi, file encryption, VPNs, TLS, government/military systems.

This app: The web UI uses AES-CBC with PKCS#7 padding; a random 16-byte IV is prepended to the ciphertext (first 32 hex digits).

Directions

  • Text: Encrypt accepts UTF-8 plaintext. Decrypt expects hex (IV + ciphertext from encrypt).
  • Key: Hex string length must match key size: 32 digits (128-bit), 48 (192-bit), 64 (256-bit).
  • File: Raw bytes in and out; use the download link after processing.
