3DES

Directions For 3DES encyptor/decryptor

For text input, only hex will be accepted. The keys are expected to be 16 character hex numbers (8 bytes). If a key greater than 8 bytes is entered, only the last 8 bytes will be used. DES encryption only works on blocks of 8 byte input. Thus, PKCS #5 padding is implemented to ensure that the input is always a multiple of 8 bytes. In the decryption step, if this padding is not respected and correctly implemented, this implementation will not be able to decrypt the message/file.

3DES Algorithm Description

PKCS #5 Padding Implemented

3DES works by performing DES 3 times. With 3 keys, this works by encrypting with key 1, decrypting with key 2, and encrypting with key 3. The reverse happens for decryption
