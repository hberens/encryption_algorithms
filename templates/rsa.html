RSA description

Type: Asymmetric cipher.

Strength: Strong when large keys are used; small keys (like this demo) are easy to break.

How it works: Generate a key pair in-app (random primes or chosen p/q), or upload e and d and provide n. Encrypt: C ≡ M^e (mod n). Decrypt: M ≡ C^d (mod n). Security rests on the difficulty of factoring n.

Key size: 2048–4096 bits are typical in production. This page uses tiny primes for speed.

Files: You can encrypt/decrypt any binary file byte-by-byte. Ciphertext is comma-separated integers (text). Key file can provide n, e, and d (JSON or lines like n=..., e=..., d=...).
